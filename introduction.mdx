## Getting Started with Self

Self Documentation is your comprehensive platform for organizing and managing all your project documentation in one centralized space. Whether you're building APIs, maintaining codebases, or creating user guides, Self provides the tools to keep everything structured and accessible.

Self streamlines documentation workflows by offering a unified interface where teams can create, update, and collaborate on docs without the complexity of scattered files or inconsistent formats.

## Key Features

<Tabs>
  <Tab title="Organization" icon="folder">
    Create hierarchical documentation structures that mirror your project's architecture. Use folders, tags, and search filters to find information quickly.
    
    <CodeGroup tabs="JavaScript,Python">
      ````javascript
      const project = {
        name: "My Documentation Project",
        folders: [
          { name: "API Docs", docs: [] },
          { name: "User Guides", docs: [] }
        ],
        tags: ["api", "guide"]
      };
      ````
      ````python
      project = {
        "name": "My Documentation Project",
        "folders": [
          {"name": "API Docs", "docs": []},
          {"name": "User Guides", "docs": []}
        ],
        "tags": ["api", "guide"]
      }
      ````
    </CodeGroup>
  </Tab>
  <Tab title="Version Control" icon="git-branch">
    Track changes over time with built-in versioning that preserves document history and allows rollback to previous states.
  </Tab>
  <Tab title="Search & Discovery" icon="search">
    Powerful search functionality helps users find relevant documentation instantly across all projects.
  </Tab>
</Tabs>

## Quick Setup

<Steps>
  <Step title="Create Account" icon="user">
    Sign up for a Self Documentation account to get started with your first project.
  </Step>
  <Step title="Set Up Project" icon="folder">
    Create a new documentation project and define its structure.
    
    <CodeGroup tabs="CLI,API">
      ````bash
      self init --project "My Project" --template basic
      ````
      ````javascript
      const response = await fetch('/api/projects', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({
          name: "My Project",
          template: "basic"
        })
      });
      ````
    </CodeGroup>
  </Step>
  <Step title="Add Documentation" icon="file-text">
    Start adding your markdown files and organize them into appropriate folders.
  </Step>
  <Step title="Invite Team" icon="users">
    Collaborate by inviting team members and setting appropriate permissions.
  </Step>
</Steps>

<Callout kind="tip">Tip: Start with a small project to familiarize yourself with Self's interface before scaling to larger documentation initiatives.</Callout>

## Use Cases

Self Documentation serves various scenarios:

- **API Documentation**: Generate and maintain comprehensive API docs from OpenAPI specifications.
- **Product Guides**: Create user manuals, tutorials, and onboarding materials.
- **Internal Knowledge Base**: Build a centralized repository for team processes and best practices.
- **Project Documentation**: Track requirements, designs, and implementation details.

<Expandable title="Advanced Configuration" default-open="false">
  For advanced users, Self supports custom themes, integrations with CI/CD pipelines, and automated publishing workflows. Configure webhooks to trigger updates when your codebase changes, ensuring documentation stays current.
  
  <CodeGroup tabs="YAML,JSON">
    ````yaml
    webhooks:
      - event: "code_push"
        action: "update_docs"
        target: "/api/docs"
    ````
    ````json
    {
      "webhooks": [
        {
          "event": "code_push",
          "action": "update_docs",
          "target": "/api/docs"
        }
      ]
    }
    ````
  </CodeGroup>
</Expandable>
